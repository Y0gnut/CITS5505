AI Reflection Log

A critical evaluation of how AI tools and large language models were used during this project, including example prompts, outputs, corrections made, and limitations observed.

Case 1: AI CSS Debugging — Treating Symptoms, Not the Cause

Prompt Used

"The text in the hero section is not visible against the background image. Fix the text colour."

What AI Produced

AI gave multiple sequential fixes — first adding the .info-card class, then adding inline styles, then adding !important to force override. Each solution failed.

What I Changed

After repeated failed attempts, I manually inspected the HTML structure and discovered the hero section was not inside the .page-content container, so the CSS selectors were never targeting the right element. The AI's fixes were applied to the wrong scope entirely.

Critical Evaluation

AI debugging tends to be symptomatic rather than diagnostic. It cannot see the browser's rendered output or computed styles, so it guesses based on common patterns. This session demonstrated that AI has no awareness of the actual DOM structure at runtime. A human developer needs to provide precise context — or better yet, use browser DevTools — rather than relying on AI to identify root causes independently.

Case 2: AI Fulfilled the Brief But Missed the Intent

Prompt Used

"Create a quiz page with at least 10 questions loaded from a JSON file."

What AI Produced

AI created exactly 10 questions in the JSON file. Technically, this satisfied the brief. However, the actual intent was to have a larger question pool (20 questions) that randomly selects 10 each session, so that repeat attempts feel different and the quiz has more variety.

What I Changed

I explicitly rewrote the prompt to specify 20 questions in the JSON file and instructed the JS to randomly select 10 per session using .slice(0, 10) after shuffling.

Critical Evaluation

This highlights a fundamental limitation of AI: it processes the literal meaning of language, not the underlying intent. The brief said "at least 10 questions" and AI delivered exactly 10 — correct, but uninspired. This mirrors a broader limitation: AI struggles to infer what a person actually wants beyond what they literally said. Interestingly, this is something humans also struggle with when communicating with each other — but the gap feels more pronounced with AI because it has no shared lived experience to draw from, only statistical patterns in text.

Case 3: AI Image Generation and the Risk of Unintentional Plagiarism

Prompt Used

"Generate a background image in the style of this image [reference image found online]."

What AI Produced

The AI (Nano Banana) returned an image that was effectively a colour palette swap of the reference image rather than a genuinely original design inspired by the style.

What I Changed

I used the output cautiously and modified it further, rather than using it directly.

Critical Evaluation

This raises a real ethical concern: AI image generators are probabilistic systems trained on existing images. When a prompt explicitly references a specific image, the output can closely resemble the original work, potentially infringing on the original creator's copyright — even if unintentionally. The ambiguity of the prompt ("in the style of") made this worse, as the AI interpreted it as "copy this as closely as possible." This suggests that vague prompts carry genuine legal and ethical risk when used with generative AI tools. Users have a responsibility to review AI-generated assets critically and not assume that "AI-generated" automatically means "original" or "safe to use."
