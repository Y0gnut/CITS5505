HTML: Building the Structure of a Web Page

HTML is the skeleton of every web page. It defines what is on the page — not how it looks or behaves.

What is HTML?

HTML stands for HyperText Markup Language. You use it to describe the content of a page using tags. Each tag tells the browser what a piece of content is — a heading, a paragraph, a link, an image, and so on. 

<!-- A basic HTML page -->
<!doctype html>
<html lang="en">
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>This is my first web page.</p>
  </body>
</html>

The <head> section holds information about the page (like its title). The <body> section holds everything the user actually sees.

Headings and Paragraphs

HTML has six levels of headings, from <h1> (most important) to <h6> (least important). Use them in order — do not skip from <h1> to <h4> just for a bigger font size. That breaks the page structure and hurts accessibility. 

<h1>Page Title</h1>
<h2>Section Heading</h2>
<h3>Sub-section Heading</h3>
<p>A paragraph of text goes here.</p>

Links and Images

Links use the <a> tag with an href attribute pointing to the destination. Images use <img> with a src (file path) and alt (description) attribute. The alt text is read aloud by screen readers and shown when the image cannot load — always include it. 

<!-- A clickable link -->
<a href="https://example.com">Visit Example</a>

<!-- An image with alt text -->
<img src="photo.jpg" alt="A photo of the campus" />

Why Use Semantic Tags?

You could wrap everything in <div> tags and it would still display. But browsers, search engines, and screen readers do not know what each <div> means. Semantic tags like <header>, <nav>, <main>, and <footer> give the page meaningful structure. 

<!-- Without semantics: hard to understand -->
<div>
  <div>My Website</div>
  <div>Home | About | Contact</div>
</div>

<!-- With semantics: clear and meaningful -->
<header>
  <h1>My Website</h1>
  <nav>Home | About | Contact</nav>
</header>

The semantic version tells the browser "this is the page header" and "this is a navigation menu." This improves accessibility for users with screen readers and helps search engines understand your content.

Lists

Use <ul> for unordered (bullet) lists and <ol> for ordered (numbered) lists. Each item goes inside a <li> tag. 

<!-- Unordered list -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list -->
<ol>
  <li>Open the file</li>
  <li>Edit the content</li>
  <li>Save and refresh</li>
</ol>

CSS: Styling and Laying Out a Web Page

CSS stands for Cascading Style Sheets. It controls how HTML elements look — colours, fonts, spacing, and layout.

How CSS Works

A CSS rule has two parts: a selector that targets an HTML element, and a declaration block that sets its style properties. 

/* Select all p tags and make the text blue */
p {
  color: blue;
  font-size: 1rem;
}

Selectors

Selectors tell CSS which elements to style. The three most common types are: 

/* Tag selector — targets every h1 */
h1 {
  color: navy;
}

/* Class selector — targets any element with class="highlight" */
.highlight {
  background-color: yellow;
}

/* ID selector — targets the one element with id="logo" */
#logo {
  width: 120px;
}

The Box Model

Every HTML element is a rectangular box. The box model describes the space around content: padding is inside the border, border is the edge, and margin is outside the border. Use box-sizing: border-box so padding and border are included in the element's total width — this prevents unexpected overflow. 

.card {
  box-sizing: border-box;
  width: 300px;
  padding: 1rem;
  border: 2px solid #333;
  margin: 1rem;
}

Responsive Design with Media Queries

A responsive page adapts its layout to different screen sizes. Media queries let you apply different styles depending on the viewport width. Start with mobile styles first, then add rules for larger screens. 

/* Default: single column on mobile */
.container {
  display: flex;
  flex-direction: column;
}

/* At 768px and wider: switch to two columns */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}

JavaScript: Making Pages Interactive

JavaScript is the programming language of the web. It controls how a page behaves — responding to clicks, updating content, and communicating with servers.

Variables and Data Types

Variables store values. Use const for values that never change, and let for values that might change later. Avoid var — it has confusing scoping rules that can cause hard-to-find bugs. 

const siteName = "CITS5505"; // never reassigned
let score = 0;               // will change as the quiz runs

score = score + 1;           // update the score
console.log(score);          // 1

Functions

A function is a reusable block of code. Define it once, call it anywhere. Functions help keep your code organised and avoid repetition. 

// Define a function
function greet(name) {
  return "Hello, " + name + "!";
}

// Call the function
console.log(greet("Alex")); // Hello, Alex!

Selecting and Changing Elements

JavaScript can find HTML elements on the page using document.querySelector() and then change their content, style, or attributes. 

// Find the element with id="message"
const box = document.querySelector("#message");

// Change its text content
box.textContent = "You clicked the button!";

Responding to Events

Events fire when the user does something — clicks a button, types in a field, or submits a form. Use addEventListener to run code when an event happens. 

const btn = document.querySelector("#myButton");

btn.addEventListener("click", function () {
  alert("Button was clicked!");
});

Fetching Data with AJAX

AJAX lets JavaScript load data from a file or server in the background — without reloading the page. The modern way to do this is with fetch() and async/await. Always wrap it in try/catch to handle errors gracefully. 

async function loadData() {
  try {
    const response = await fetch("data/questions.json");
    const data = await response.json();
    console.log(data); // use the data
  } catch (error) {
    console.error("Failed to load:", error);
  }
}

loadData();
Key Takeaway: JavaScript controls how your page behaves. Use it to respond to user actions, update the page without reloading, and load data from external sources with AJAX.
